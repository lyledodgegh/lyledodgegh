GitHub Account Types

Personal Accounts vs. Enterprise Managed Users (EMU)

👤

Personal Account github.com

Self-managed, user-owned identity

🆔 Identity
🙋
User creates & ownsSigned up independently at github.com with any email
🔑
Login managed by userPassword, 2FA, passkeys — user's responsibility
✏️
Username is flexibleAny available handle, e.g. @jane-dev
🏠 Ownership
🔒
Account belongs to the userPersists even if they leave a company
🌍
Can contribute anywherePublic repos, open source, personal projects, side jobs
🏢 Org Access
📨
Invited to orgsEmployer invites the personal account; user can accept or decline
⚠️
Audit gap riskCompany can't control account creation or force MFA at account level
🏛️

Enterprise Managed User EMU

Company-provisioned, IT-controlled identity

🆔 Identity
🏢
Enterprise creates itProvisioned via IdP (Okta, Entra ID, etc.) using SCIM
🔐
Login via SSO onlySAML / OIDC — no separate GitHub password
🏷️
Username is enforcedFormat: jane_companyslug — set by IdP
🏠 Ownership
🔒
Account belongs to employerSuspended or deleted when employee offboards — automatically
🚧
Walled gardenCannot contribute to external orgs, fork to personal namespace, or access public repos with the EMU account
🏢 Org Access
⚙️
Auto-provisioned to orgsIdP group membership → GitHub org/team membership automatically
✅
Full enterprise controlAdmins enforce policy: IP allow-lists, required 2FA, content restrictions
